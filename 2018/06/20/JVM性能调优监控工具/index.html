    以下是一些JVM性能调优监控工具


    jps、jstack、jmap、jhat、jstat、hprof….

    对于java程序,有时可能会碰到以下问题
    • OutOfMemoryError，内存不足
    • 内存泄露
    • 线程死锁
    • 锁争用（Lock Contention）
    • Java进程消耗CPU过高
    • ….

    通过一些监控工具去对java程序进行调优以及监控,在发生问题时使用工具去进行定位问题为后续的排障、修复问题提到了很好的辅助作用。

    jps主要用来输出JVM中运行的进程状态信息。语法格式如下：

    jps [options] [hostid]

    如果不指定hostid就默认为当前主机或服务器。

    命令行参数选项说明如下：


    -q 不输出类名、Jar名和传入main方法的参数
    -m 输出传入main方法的参数
    -l 输出main类或Jar的全限名
    -v 输出传入JVM的参数

    example:


    root@ubuntu:/# jps -m -l
    2458 org.artifactory.standalone.main.Main /usr/local/artifactory-2.2.5/etc/jetty.xml
    29920 com.sun.tools.hat.Main -port 9998 /tmp/dump.dat
    3149 org.apache.catalina.startup.Bootstrap start
    30972 sun.tools.jps.Jps -m -l
    8247 org.apache.catalina.startup.Bootstrap start
    25687 com.sun.tools.hat.Main -port 9999 dump.dat
    21711 mrf-center.jar

    jstack

    jstack主要用来查看某个Java进程内的线程堆栈信息。语法格式如下：



    jstack [option] pid
    jstack [option] executable core
    jstack [option] [server-id@]remote-hostname-or-ip
        命令行参数选项说明如下：

    -l long listings，会打印出额外的锁信息，在发生死锁时可以用jstack -l pid来观察锁持有情况
    -m mixed mode，不仅会输出Java堆栈信息，还会输出C/C++堆栈信息（比如Native方法）

    jstack可以定位到线程堆栈，根据堆栈信息我们可以定位到具体代码，所以它在JVM性能调优中使用得非常多。下面我们来一个实例找出某个Java进程中最耗费CPU的Java线程并定位堆栈信息，用到的命令有ps、top、printf、jstack、grep。

    系统查看top命令，定位CPU异常进程。
    进入对应容器内，查看异常线程：
    进入容器：


    sudo docker exec -ti space_space_1 /bin/bash -l

    查看线程情况（在容器内执行）：


    export TERM=xterm
    ps auxf     # 定位进程在容器内的PID，对空间来说数字1。
    top -H -p 1 # -H 查看线程ID，记录下占用CPU最多的线程ID数，比如是：43，转为16进制：0x2B，再转成小写模式：0x2b

    对java进行jstack（退出容器，在系统中执行和查看）：


    sudo docker exec -u docker -ti space_space_1 /home/jdk/bin/jstack 1 > jstack.log

    有时java会没有响应，可加 -F 参数：

    sudo docker exec -u docker -ti space_space_1 /home/jdk/bin/jstack -F 1 > jstack.log

    还有时 -F 参数也没有效果，就只能多试几次了。

    查看jstack.log文件，并搜索上面记录的小写16进制的线程ID：nid=0x2b


    “GC task thread#0 (ParallelGC)” os_prio=0 tid=0x00007fb9e401f800 nid=0x2b runnable

    根据jstack.log中的线程ID，判断：
    如果是GC线程问题，那还需进行内存信息抓取。
    如果是其他线程问题，将整个线程信息提交给开发人员。
